# 교합면 기반 치아 세그멘테이션 파이프라인 설계

> Claude Code 작업 인계용 문서. 구강 스캔 메시(IOS, OBJ/PLY/STL)에서 개별 치아를
> 인스턴스 단위로 분할하기 위한 coarse-to-fine 하이브리드 파이프라인의 설계 결정과
> 구현 시 주의점을 정리한다.

---

## 1. 목표와 범위

- 입력: 단일 아치 구강 스캔 표면 메시 (상악 또는 하악, OBJ/PLY/STL).
- 출력: 치아 인스턴스 분할(정점별 라벨) + 위치(localization). 가능하면 FDI 치식 번호.
- **범위 밖**: 마진 라인 검출은 별개의 다운스트림 문제다. 마진은 치은 경계 또는
  치은연하(subgingival)에 있어 교합면 정수리 뷰에서 보이지 않는다. 이 파이프라인은
  "세그멘테이션/위치"를 풀고, "마진"은 풀지 않는다. 범위를 흐리지 말 것.

이 구조는 풀 메시를 통째로 GCN/Point Transformer에 넣는 end-to-end 대비 가볍고,
통제 가능하며, 디버깅이 쉽다. 대신 단계가 많아 **앞 단계 실패가 뒤로 전파되는
error propagation 위험**이 가장 큰 약점이다. 초기 투자를 1단계(평면 추정)와
2단계(멀티채널 렌더)에 집중한다.

이 접근은 SOTA의 coarse-to-fine 철학과 동일하다. 참고: TSegLab은 2D CNN으로 거친
위치 검출 후 곡률 기반 3D 분할을 수행하는 2단계 구조로 Teeth3DS에서 SOTA를 기록했다.

---

## 2. 전체 파이프라인

```
[메시 입력]
   |
   v
(1) 교합면 추정 (PCA 부트스트랩 + 교두정 정제)
   |   - 전체 메시 PCA로 거친 교합축 추정
   |   - 교두정 후보 추출 (높이 + 곡률 + 격자 NMS)
   |   - 교두정만으로 PCA 재실행 -> 평면 정제
   |   - 법선 부호(+/-) 확정
   v
(2) 멀티채널 정사영 렌더 캡처
   |   - 교합 법선 방향 orthographic projection
   |   - 채널: depth(교합축 Z-buffer) + normal + curvature
   v
(3) 2D rough 인스턴스 세그멘테이션 -> 치아별 AABB
   |   - instance seg / detector 사용 (semantic + CC 금지)
   v
(4) AABB 확장 + 교합축 기둥(column) back-projection -> 메시 ROI 추출
   |   - 깊이 범위 클리핑 필수
   v
(5) ROI 내 메시 정밀 분할 (곡률 기반 graph-cut / region growing)
   |   - 교두정을 시드로 재활용
   v
[치아 인스턴스 분할 결과]
```

## 3. 단계별 설계 결정과 주의점

### 단계 1. 교합면 추정 — PCA 부트스트랩

**원리**: 아치는 폭(좌우) 분산이 가장 크고, 수직(교합-치은) 분산이 가장 작다.
따라서 최소 고윳값에 해당하는 제3주성분이 교합면 법선이다.

**전체 메시 단일 PCA의 문제**: 구개(palate), 확장된 치은, 베이스 트림 등 비교합
정점이 분산에 끼어들어 평면을 기울인다. 결손치/부분 치열이면 아치 비대칭으로 또
틀어진다. 실제 교합면은 평면이 아니다(전후 Curve of Spee, 좌우 Curve of Wilson).

**해결: 닭-달걀 문제를 부트스트랩으로 푼다.**
교두정은 "교합 법선 방향 국소 극점"인데, 그 법선을 구하려고 교두정을 찾으니 순환.
1. 전체 메시 PCA로 거친 교합축을 한 번 잡는다.
2. 그 축 기준으로 교두정 후보를 뽑는다.
3. 후보들로 PCA를 다시 해 평면을 정제한다.
4. 1~2회 반복하면 수렴한다.

**교두정 후보 추출 (조합 권장)**:
- 거친 교합축 높이 기준 상위 점 (단독 사용 금지 — 기운 아치에서 한쪽만 뽑혀 편향)
- 곡률 필터: 교두정은 두 주곡률이 모두 양(+)인 볼록 봉우리. 양의 가우시안 곡률 +
  봉우리 형태로 절단연/능선과 구분. 곡률은 노이즈에 민감하므로 사전 스무딩 필수.
- 격자 NMS: 교합축 2D 평면에 투영 후 격자 셀별 최고점 또는 non-maximum suppression.
  "치아당 한두 개씩 고르게" 분포시켜 PCA 입력 편향을 줄인다.

**남는 취약 케이스**:
- 전치부는 교두가 아니라 절단연(edge)이라 봉우리 검출에서 누락 가능 -> 전방 영역이
  통째로 빠지면 평면이 후방으로 치우침. 교두 정의를 "봉우리"에서 "교합축 방향 국소
  극점"으로 넓혀 절단연도 포함시킬 것.
- 마모치/보철물: 교두 평탄화로 봉우리 신호 약화.
- 결손치/부분 맹출치: 해당 영역 교두 부재.
- Curve of Spee/Wilson: 교두정이 진짜 평면이 아닌 완만한 곡면 위에 있어, PCA 평면은
  최소제곱 근사일 뿐. 거친 정렬엔 충분하나 정밀 기준면으론 한계.

**법선 부호 모호성 (놓치기 쉬움)**: PCA는 축만 주고 어느 쪽이 "위(교합)"인지 모른다.
교두정 무게중심이 전체 메시 무게중심보다 법선 양의 방향에 있도록 부호를 확정한다.
이 처리가 빠지면 렌더 캡처가 상하 반전되어 나온다.

### 단계 2. 멀티채널 정사영 렌더 캡처

**가장 중요한 결정: 단일 실루엣/색 이미지 금지.** depth(교합축 Z-buffer) + normal +
curvature를 멀티채널로 함께 렌더한다. 치아-치은 경계와 인접치 사이 골(interproximal
valley)은 곡률 급변 지점이라, 곡률 채널이 2D 검출 정확도를 크게 올린다.

**구조적 한계 인지**: 정수리 정사영은 협면/설면 같은 수직 벽면을 얇은 띠로 압축한다.
그런데 가장 모호한 경계(인접 접촉면, 치은 마진)가 바로 그 수직면에 있다. 따라서
단일 교합 뷰는 정밀 경계 추출엔 정보 부족.
-> 거친 AABB 검출 목적이면 OK. **정밀 경계는 2D 마스크에 의존하지 말고 5단계
메시에서 복원**한다. 이 원칙을 고정한다.

### 단계 3. 2D rough 세그멘테이션 -> AABB

**패러다임 선택이 핵심**: semantic seg 후 connected component로 AABB를 뽑으면 접촉이
빡빡한 인접치가 하나로 병합된다. **인스턴스 세그(Mask R-CNN, YOLO-seg) 또는
검출기**로 치아 단위 분리. 결손치/gap 케이스를 학습 데이터에 반드시 포함해야
검출기가 빈 공간을 합치거나 메우지 않는다.

### 단계 4. AABB 확장 + 메시 ROI 추출

치아 기울기로 치관 풋프린트가 실제 범위보다 작고, 인접/치은 맥락이 필요하므로 확장은
타당. AABB는 이미지축 정렬이라 곡선 아치 구치부에서 이웃 치아를 상당히 포함하는데,
"맥락 포함" 목적상 허용 가능. 단 5단계가 타깃과 이웃을 확실히 분리해야 함. 더 타이트하게
가려면 아치 접선 방향 OBB(oriented bounding box)도 선택지.

**back-projection 주의**: 확장 박스를 교합축 방향 기둥(column)으로 back-project해
메시 정점/면을 ROI로 자를 때, 심한 경사치 오버행이나 상하악 동시 스캔이면 기둥이
엉뚱한 것을 물 수 있다. **깊이 범위 클리핑 필수.**

### 단계 5. ROI 메시 정밀 분할

ROI가 한 치아 중심으로 작게 잘려 있어 무거운 네트워크 불필요. 검출된 치아 중심
(또는 교두정)을 시드로 region growing 하되, **인접면 골과 치은구(gingival groove)의
곡률 능선/골에서 멈추는** graph-cut 또는 harmonic field 기반 분할. (TSegLab이 곡률·
하모닉 매핑으로 한 것과 같은 발상.) 작은 ROI 덕에 학습 기반/기하 기반 둘 다 현실적.

## 4. 데이터 / 평가 (참고)

**공개 데이터셋**: Teeth3DS+ (3DTeethSeg 2022 / 3DTeethLand 2024 MICCAI 챌린지).
900명 환자, 1,800 스캔, 23,999 주석 치아. Primescan/Trios 3/iTero로 수집, OBJ +
정점별 인스턴스/클래스 주석(JSON). 교정 50% / 보철 50% 균형.
- 한계: 교정기/임플란트/광범위 수복 과소 대표, 치관/치은 중심(치근 X).
- 알려진 오류 상관: 잔존 치근, 잔존 치관, 결손치, 부분 맹출치에서 모델 오류 증가.
- 자체 임상 데이터로 위 취약 케이스 보강 필요.

**평가 지표 (설계 전 합의)**: TLA(위치), TSA(세그), TIR(식별) + 정점 Dice / macro-F1
/ macro-IoU. CAD 품질엔 **경계 정밀도(Hausdorff, boundary-F1)**를 별도 지표로.
치아-치은 경계 = 마진 라인 품질 직결이므로 평균 Dice만 보지 말 것.

## 6. 권장 구현 순서 (마일스톤)

1. 메시 로딩 + 전처리(스무딩, 법선/곡률 계산) 유틸 정비.
2. **단계 1 부트스트랩 교합면 추정** 구현 (전체 PCA -> 교두정 검출 -> 재PCA -> 부호 확정).
   - 여기가 전 파이프라인의 토대. 시각화로 평면/법선/교두정을 먼저 검증.
3. 단계 2 멀티채널 렌더 캡처 (depth/normal/curvature).
4. 단계 3~4 2D 검출 -> AABB 확장 -> ROI back-projection (깊이 클리핑 포함).
5. 단계 5 ROI 곡률 기반 분할.
6. 평가 하니스(TLA/TSA/TIR + 경계 지표) 연결, Teeth3DS+로 벤치마크.

> 구현 규약: 긴 함수는 작은 named helper로 분리하고 각 함수는 완결된 본문으로 작성.
> 코드 주석에 유니코드(한글 등) 사용 금지(ASCII 주석만). 임시방편/땜질식 해결 금지,
> 근본 원인 수정 우선.

## 7. 한 줄 요약

PCA로 교합면을 잡되 **전체 메시 단일 PCA가 아니라 교두정 추출 후 재PCA하는 부트스트랩**
구조가 핵심. 평면 추정 정밀화(단계1)와 멀티채널 렌더(단계2)에 투자하고, 정밀 경계는
2D가 아닌 메시(단계5)에서 복원한다. 마진 검출은 범위 밖.
